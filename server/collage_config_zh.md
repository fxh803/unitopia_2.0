# Unitopia 配置文件说明（dsl.json）

本文件介绍 Unitopia 拼贴系统的配置文件（`dsl.json`）格式及各字段含义。请根据实际需求进行填写。

## 顶层结构

```json
{
  "collage": [
    { ... }, 
    { ... }
  ]
}
```
- `collage`：拼贴任务列表，每个元素为一个拼贴任务对象。

---

## 拼贴任务对象

每个拼贴任务包含以下字段：

| 字段名             | 类型      | 说明                                                         |
|--------------------|-----------|--------------------------------------------------------------|
| collage_id         | int       | 拼贴任务编号（唯一标识）                                     |
| iterations         | int       | 优化迭代次数，影响生成效果 ，可选                                   |
| render_size         | int       | 画布大小，可选                                   |
| marker_config      | list      | 标记（元素）配置列表                                         |
| container_config   | dict      | 容器配置，定义拼贴区域                                       |
| emitter_config     | dict      | 发射器配置，定义元素初始分布  ，可选                                |
| force_config       | dict      | 力场配置，定义元素受力方式   ，可选                                 |

---

### marker_config（标记配置）

每个 marker_config 对象包含：

| 字段名           | 类型      | 说明                                                         |
|------------------|-----------|--------------------------------------------------------------|
| marker           | list      | 标记文件路径列表（SVG/PNG），如 `["./svg/1.svg", ...]`       |
| data            | list      | 可视化数据列表                                                |
| visual_encoding  | list      | 可视编码配置，见下表                                         |
| init_size_ratio  | float     | 初始尺寸与容器的占比(0~1)，可选                                    |
| variance  | float     |    可视化图元的差异度(0~1),（越大差异度越小）                                    |
| margin  | int     |    图元间隔                             |
|   rotation   | boolean     |    是否旋转，可选                             |
#### visual_encoding 示例

```json
[
  {
    "attribute": "duration (hours)", # 数据属性名(可选)
    "channel": "size"
  }
]
```
- `attribute`：数据属性名
- `channel`：映射通道（如 size, height, width）

---

### container_config（容器配置）

| 字段名     | 类型   | 说明                                                         |
|------------|--------|--------------------------------------------------------------|
| container  | str    | 容器掩码图片路径（PNG），如 `"./mask/container.png"`。为黑白图，黑色区域为目标区域         |
| boundary   | str    | 边界类型，`"open"` 或 `"closed"` open属性时将应用力场属性（默认重力）                           |
| holes      | int/null |  指定为某拼贴 id 时，目标区域将排除该 id 的拼贴结果作为“洞”；null 表示无洞     |

---

### emitter_config（发射器配置）

| 字段名   | 类型   | 说明                                                         |
|----------|--------|--------------------------------------------------------------|
| emitter_type  | str    | 发射类型（1D/2D/空）1D意味着图元将在这根发射器上水平排布并紧紧吸附在线上，2D在1D的规则下允许数据相近的图元在法线方向上排布并紧紧吸附在线上。空意味着图元只会初始化在发射器上。         |
| control_points  | list    | 多段三阶贝塞尔曲线列表 ，用来表示发射器曲线（如下）。

### force_config（力场配置）

| 字段名   | 类型   | 说明                                                         |
|----------|--------|--------------------------------------------------------------|
| force_type    | str    | 力场类型，可选项："down"、"indicate_direction"、"points"  |
| force_points    | list    | 力场参数。在indicate_direction模式下形如[x1,y1,x2,y2]，表示单位向量。向量方向即力场方向。在points 模式下形如[ [x1,y1],[x2,y2] ]  表示多个力吸引点         |

## 示例配置

```json
{
  "collage": [
    {
      "collage_id": 1,
      "marker_config": [
        {
          "marker": [
            "./example/marker.svg"
          ],
          "visual_encoding": [
            {
              "channel": "size",
              "attribute": "percentage_of_growth"
            }
          ],
          "margin": 5,
          "data": [
            "1.04",
            "1.34",
            "1.31",
            "1.75",
            "1.01",
            "1.39",
          ],
          "init_size_ratio":0.2,
          "variance": 0.5
          
        }
      ],
      "container_config": {
        "container": "./example/container.png",
        "boundary": "closed",
        "holes": null
      },
      "emitter_config": {  
        "control_points" : [   
          [0.3252644230769231, 0.19846153846153845],
          [0.3252644230769231, 0.05017085146823048],
          [0.7711396995269381, 0.21629313100576172],
          [0.6098798076923077, 0.2969230769230769],
          [0.5773568103595865, 0.3131845755894375],
          [0.5406153325688164, 0.32185963620324264],
          [0.50680288461538464, 0.33538461538461536],
          [0.4620191099846794, 0.35329812523689753],
          [0.417768179496137, 0.3607022065457396],
          [0.3791105769230769, 0.3907692307692308],
          [0.18812241021289742, 0.5393155826549258],
          [0.7114080620299861, 0.5742741498340974],
          [0.6760336538461538, 0.7246153846153846],

        "force_type": "points",
        "force_points": [[0.5,0.5]]
      }
    }
  ]

```

---

## 注意事项

- `marker` 字段需填写 SVG 或 PNG 文件路径，不能为空。
- `container` 字段建议填写 PNG 掩码文件路径，若为空则为默认矩形区域。
- 详细字段说明及更多高级用法请参考主项目文档或联系开发者。
